[JavaScript] 마이크로 태스크큐, 매크로 태스크큐

2023-09-02

예전에 프론트엔드 분야에서 잘나가는 친구한테 기술 질문 몇 개 해달라고 했었는데

그 때 나온 질문 중 하나가 마이크로 태스크큐와 매크로 태스크큐의 차이에 대한 질문이었다.

당시 나는 태스크큐가 더 세분화될 수 있는 거였는지 몰랐었고

찾아봤더니 동작 방식이 다르고, 함수에 따라 나뉜다고 나와있었다.

그 내용은 다음과 같다.

마이크로 태스크 큐 (Micro Task Queue)매크로 태스크 큐 (Macro Task Queue)
공통점- 선입선출(FIFO, First In First Out) 구조- 이벤트 루프에 의해 실행- 선입선출(FIFO, First In First Out) 구조- 이벤트 루프에 의해 실행
차이점- Promise, process.nextTick, queueMicrotask, MutationObserver
  • 매크로 태스크 큐의 작업보다 먼저 실행
  • 큐에 새로운 작업이 들어오더라도 큐가 빈 상태가 될 때까지 계속해서 실행 | - requestAnimationFrame, I/O, UI rendering, setTimeout, setInterval, setImmediate
  • 마이크로 태스크 큐의 작업들이 모두 끝난 후 실행
  • 도중에 큐에 들어온 새로운 작업은 다음 이벤트 루프가 실행될 때까지 실행하지 않음 |

……

저 표는 나중에 보자. 개발자는 코드로 이야기한다.

다음 코드를 보고 결과를 예상해보자. /api에 요청하여 받는 응답은 'data!!' 라는 문자열이라 가정한다.

function main() {
    console.log("start");
    
    setTimeout(function () {
    	console.log("start api");
        fetch("/api").then(function (response) {
            setTimeout(function () {
                console.log(response.data)
            }, 0);
        });
        console.log("end api");
    }, 0);
    
    console.log("end");
}

main();
  1. 스크립트를 실행하는 작업이 먼저 매크로 태스크 큐에 추가되고, 이벤트 루프는 이 작업을 먼저 실행한다.

  1. 마지막 라인의 main()이 실행되고 콜 스택에 추가된다.

  1. 함수 main의 첫 라인이 실행되어 "start"가 출력된다.

  1. 자바스크립트 엔진이 setTimeout 함수를 보고 Web API의 Timer를 실행시켜 0초 뒤에 콜백 함수가 매크로 태스크 큐에 추가되도록 한다.

  1. 함수 main의 마지막 라인이 실행되어 "end"가 출력된다.

  1. 함수 main이 종료되어 콜 스택에서 사라지고, 브라우저는 렌더링 할 것이 남아있다면 렌더링 작업을 수행한다.

  1. 매크로 태스크 큐에 남아있던 setTimeout 콜백 함수가 실행된다.

  1. setTimeout 콜백 함수의 첫 라인이 실행되어 "start api"가 출력된다.

  1. 자바스크립트 엔진이 fetch 함수를 보고 Web API의 XMLHttpRequest를 실행시켜 응답받았을 때 실행할 then의 콜백 함수를 마이크로 태스크 큐에 추가한다.

  1. setTimeout 콜백 함수의 마지막 라인이 실행되어 "end api"가 출력된다.

  1. setTimeout 콜백 함수가 종료되어 콜 스택에서 사라지고, 마이크로 태스크 큐에 남아있던 Promise 콜백 함수를 실행한다.

  1. 자바스크립트 엔진이 setTimeout 함수를 보고 Web API의 Timer를 실행시켜 0초 뒤에 콜백 함수가 매크로 태스크 큐에 추가되도록 한다.

  1. Promise 콜백 함수가 종료되어 콜 스택에서 사라지고, 브라우저는 렌더링 할 것이 남아있다면 렌더링 작업을 수행한다.

  1. 매크로 태스크 큐에 남아있던 setTimeout 콜백 함수가 실행되면서, API 호출 시 응답받은 데이터("data!!")가 출력된다.
