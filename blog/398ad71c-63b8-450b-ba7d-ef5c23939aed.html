Vue.js로 개발 시 성능 이슈에 대한 두 번째 이야기를 하려 한다.

회사 프로젝트 개발 도중에 API로 받은 데이터를 표출하는 데 굉장히 오래걸렸던 적이 있었다.

API 실행 자체는 오래걸리지 않았기에 렌더링에 문제가 있을거라 생각했고,

구글링을 해보니 원인은 '__ob__' 프로퍼티가 있는 데이터들이었다.

이 프로퍼티는 Vuex에 Mutation할 데이터가 리스트나 객체일 경우

그걸 그대로 할당해버릴 때 생긴다고 한다.

리스트의 경우 다음과 같이 되어버린다.

export default createStore({
    state: {
    	data: null
    },
    mutation: {
    	SET_DATA: (state, newData) => state.data = newData	// [1, 2, 3]
    },
    ...
});

console.log($store.state.data);

위의 두 이미지는 실제로 진행했던 프로젝트의 개선 전의 모습이다.

API로 응답받은 리스트의 각 요소에 get, set, 그리고 __ob__ 등 원치 않는 프로퍼티까지 포함되어 있다.

이것은 Vue.js가 내부적으로 initData를 호출할 때 발생하는데,

변경감지를 위해 데이터에 Observer 및 getter와 setter를 자동으로 할당해주게 된다.

하지만 나는 Vuex를 단지 상태 관리 용도로만 쓰고 싶었기 때문에 열심히 해결책을 찾아다녔고,

다음과 같은 방법으로 이 문제를 해결하였다.

export default createStore({
    state: {
    	data: null
    },
    mutation: {
    	SET_DATA: (state, newData) => state.data = Object.freeze(newData)	// [1, 2, 3]
    },
    ...
})

말 그대로 데이터를 '얼려버렸다'.

Object.freeze는 데이터와 그 하위 데이터까지 불변하게 만드는 자바스크립트 내장 스펙이다.

불변하게 만듦으로써 Vue에서 initData가 실행되더라도

데이터는 그대로 보존된다.
