[React Native] React Native ECharts

2023-11-24

드디어 React Native에도 ECharts 라이브러리가 등장했다.

그동안 회사 프로젝트에서 Victory Native를 쓰면서 성능 문제로 느리다는 고객 문의가 많았었는데

이번 기회로 어느 정도 개선할 수 있지 않을까 하여 접하게 되었다.

그래서 프로젝트에 이미 Victory로 구현된 차트를 새롭게 React Native ECharts로 만들어보면서

겪은 과정과 문제들을 기록하기 위해 이 글을 쓴다.

Install

먼저 패키지 설치부터 한다.

npm i @wuba/react-native-echarts echarts react-native-svg @shopify/react-native-skia      # npm
yarn add @wuba/react-native-echarts echarts react-native-svg @shopify/react-native-skia   # yarn
pnpm add @wuba/react-native-echarts echarts react-native-svg @shopify/react-native-skia   # pnpm

Android의 경우 android/app/proguard-rules.pro 파일에 다음을 추가한다.

-keep class com.shopify.reactnative.skia.** { *; }

iOS의 경우 pod install 을 해줘야 한다.

cd ios
pod install

네이티브 라이브러리가 추가/변경되었으니 나중에 배포 시에는 당연히 마켓 심사를 거쳐야 한다.

Usage

사용은 다음과 같이 하면 된다.

기존의 ECharts option을 그대로 쓸 수 있다.

import { SkiaChart, SVGRenderer } from '@wuba/react-native-echarts';
import { LineChart } from 'echarts/charts';
import { GridComponent } from 'echarts/components';
import * as echarts from 'echarts/core';
import { EChartsOption } from 'echarts/types/dist/shared';
import { useEffect, useRef } from 'react';

echarts.use([SVGRenderer, LineChart, GridComponent]);

export default function App() {
  const skiaRef = useRef<any>(null);

  const option: EChartsOption = {
    xAxis: {
	    type: 'category',
	    data: ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun']
	  },
	  yAxis: {
	    type: 'value'
	  },
		series: [
	    {
	      data: [15, 23, 22, 21, 13, 14, 26],
	      type: 'line'
	    },
		],
  };

  useEffect(() => {
    if (skiaRef.current) {
      const chart = echarts.init(skiaRef.current, 'light', {
        renderer: 'svg',
        width: 600,
        height: 400,
      });
      chart.setOption(option);
	    return () => chart.dispose();
    }
  }, [width, height, option]);

  return <SkiaChart ref={skiaRef} />;
}

ECharts를 쓰던 사람은 알겠지만 option의 series에 여러가지 차트를 추가할 수 있다.

근데 그냥 하면 안되고 다음과 같이 내가 추가하려는 차트의 종류에 해당하는 컴포넌트를

포함시켜줘야 한다.

import { SkiaChart, SVGRenderer } from '@wuba/react-native-echarts';
import { BarChart, LineChart, ScatterChart, ... } from 'echarts/charts';
import { GridComponent } from 'echarts/components';
import * as echarts from 'echarts/core';

echarts.use([
	SVGRenderer,
	LineChart,
	GridComponent,
	BarChart,
	ScatterChart,
	...
]);

Problems

역시나 문제를 마주쳤다.

우선, SkiaChart 에서 fontFamily 가 적용이 안된다.

툴팁, 축 레이블 등 어느 곳에 적용해봐도 먹히지가 않는다.

찾아봤더니 아직 지원 안한다고 한다. (Well-known Issue)

그리고 iOS에서는 한글이 깨진다…

locale 설정이 필요한건지 해서 검색해봤더니 SvgChart 를 쓰라고들 한다.

그리고 툴팁의 backgroundColor 의 alpha 값으로 투명도를 주면

SkiaChart 에서 적용이 안된다.

rgba css 함수로 하면 적용됨 (해결)

series.markLine.symbol 에다가 base64 이미지 문자열을 적용하면

마찬가지로 SkiaChart 에서 안되더라.

이것도 아직 지원 안된다고 한다. (Well-known Issue)

이 문제들이 과연 이슈일지, 아니면 내가 원인이 되어 나타난 문제인지는 좀 더 파악을 해봐야겠다.

Compare

이 라이브러리에서는 SkiaChart 와 SvgChart 두 가지를 제공한다.

SkiaChart 는 Shopify라는 곳에서 오픈소스로 만든 고성능 그래픽 라이브러리 Skia 기반의 차트이다.

Skia 관련해서는 여기를 참고하면 좋을 듯 하다.

bookmark

SvgChart 는 react-native-svg 라이브러리 기반의 차트이다.

두 차트 컴포넌트의 차이점은 성능이다.

SkiaChart 는 내부적으로 Skia DOM API를 사용하여 JS Thread 외부에서 UI를 그린다고 한다.

하지만 SvgChart 는 기존의 React Native 렌더링 방식으로 UI가 렌더링된다.

UI Thread에서 UI를 그리다가 JS 연산이 필요할 경우

JS Thread에서 할일 다 끝내고 다시 UI Thread에 올 때까지 UI가 다 그려지는 시간이 지연된다.

직접 차트를 구현해보니 당연히 SkiaChart 렌더링이 빠르고 깔끔했고,

SvgChart 는 기존에 사용하던 Victory Native로 구현한 차트에서 발생했던 Freezing 현상이 똑같이 나타났다.

(근데 체감상 Victory 차트에서 나타났던 Freezing 시간보다는 조금 덜한 느낌)

Conclusion

사실상 위에서 언급한 문제들만 해결될 수 있다면

더할 나위 없이 훌륭한 라이브러리라고 할 수 있다.

성능도 성능이지만, 기존의 ECharts 기능을 그대로 써서 차트를 구현할 수 있다는 게

나에게는 너무 큰 혁명이었다. (그동안 너무 열악해서)
