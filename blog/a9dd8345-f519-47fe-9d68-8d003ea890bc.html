[Docker] 컨테이너 다루기

2023-09-02

도커 컨테이너를 다루는 법에 대해 적어보려 한다.

컨테이너 생성

docker create --name [컨테이너 이름] -p [외부 포트]:[컨테이너 내부 포트] -v [로컬경로]:[지정경로] -e [환경변수]=[환경변수에 들어갈 값] [이미지 이름 또는 이미지 ID]
  • --name: 컨테이너 이름 지정
  • -p, --publish: 포트번호 지정
  • -v, --volume: 로컬의 특정 경로와 컨테이너 내부의 특정 경로를 바인딩하여 파일 공유 (마치 공유 폴더랑 비슷)
  • -e, --env: 컨테이너 내부에서 사용될 환경변수 지정

컨테이너 실행

docker start [컨테이너 ID 또는 컨테이너 이름]

컨테이너 접속

여기서 접속은 두 가지 의미가 있다.

하나는 리눅스 Bash 쉘 기반 터미널로 접속하는 것이고,

다른 하나는 그냥 현재 컨테이너 자체에 접속하는 것이다.

다음은 리눅스 Bash 쉘 기반 터미널로 접속하는 커맨드다.

docker exec -it [컨테이너 ID 또는 컨테이너 이름] /bin/bash

  • -i, --interactive: 대화식 인터페이스, CLI로 접속할 수 있다.

  • -t, --tty: CLI로 접속 시 [root@???] $ 부분이 뜨도록 한다.

    이 옵션을 주지 않을 경우 빈 입력 화면만 나온다.

    (대신, 커맨드 수행은 제대로 된다.)

다음은 컨테이너 자체에 접속하는 커맨드이다.

docker attach [컨테이너 ID 또는 컨테이너 이름]

컨테이너 자체에 접속한다는 말이 좀 이상한데,,

쉬운 예를 들자면, 만약 톰캣 컨테이너로 attach를 하게 되면

톰캣의 catalina.out 로그가 실시간으로 보여지게 된다.

PostgreSQL 컨테이너로 attach를 하게 된다면

psql CLI를 실행할 수 있는 것이다.

exec와 attach의 차이점은 exec는 exit 명령어나 Ctrl + C를 통해

접속에서 빠져나와도 컨테이너가 중지되지 않는다.

하지만, attach로 접속했을 경우 Ctrl + C 등을 통해 빠져나오면

해당 컨테이너는 중지된다.

그리고 또 하나의 차이점은 attach는 단순 접속이지만,

exec는 컨테이너 접속 외에도 해당 컨테이너 내에서

사용 가능한 커맨드를 입력할 수 있다.

다음과 같은 경우라 보면 된다.

docker exec my-postgres psql

컨테이너 생성과 동시에 실행

docker run -d --name [컨테이너 이름] -p [외부 포트]:[컨테이너 내부 포트] -v [로컬경로]:[지정경로] -e [환경변수]=[환경변수에 들어갈 값] [이미지 이름 또는 이미지 ID]
  • -d, --detach: 백그라운드 실행
  • --name: 컨테이너 이름 지정
  • -p, --publish: 포트번호 지정
  • -v, --volume: 로컬의 특정 경로와 컨테이너 내부의 특정 경로를 바인딩하여 파일 공유 (마치 공유 폴더랑 비슷)
  • -e, --env: 컨테이너 내부에서 사용될 환경변수 지정

컨테이너 생성, 실행과 동시에 접속

docker run -it --name [컨테이너 이름] -p [외부 포트]:[컨테이너 내부 포트] -v [로컬경로]:[지정경로] -e [환경변수]=[환경변수에 들어갈 값] [이미지 이름 또는 이미지 ID]

실행중인 컨테이너 목록 조회

docker ps

모든 컨테이너 목록 조회

docker ps -a
  • -a, --all: 모든 컨테이너 조회

컨테이너 중지

docker stop [컨테이너 ID 또는 컨테이너 이름]

컨테이너 삭제

docker rm [컨테이너 ID 또는 컨테이너 이름]

컨테이너 재시작

docker restart [컨테이너 ID 또는 컨테이너 이름]
