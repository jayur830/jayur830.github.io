[Storybook] 프론트엔드 테스트

2023-09-02

Storybook이라는 테스트 도구를 안 지는 두 달이 됐는데,

정작 여기다 적는건 이제서야 적는다.

Setup

설치는 아래와 같이 하면 된다.

npm i -D @storybook/cli
npx sb init --builder webpack5        # 기본값 webpack5

or

yarn add -D @storybook/cli
yarn sb init --builder webpack5        # 기본값 webpack5

Usage

그리고 실행은 다음과 같이 하면 뭐 로딩 줄줄이 하면서 아래와 같이 뜨게 될 것이다.

npm run storybook

or

yarn storybook

Storybook을 설치하면 CRA와 Vite의 경우 src/stories 디렉토리가 생긴다.

거기에 예제 컴포넌트들이 몇 개 있는데, 위 이미지는 그 중 버튼 컴포넌트이다.

아래는 그 코드이다.

import React from 'react';
import { ComponentStory, ComponentMeta } from '@storybook/react';

import { Button } from './Button';

// More on default export: https://storybook.js.org/docs/react/writing-stories/introduction#default-export
export default {
  title: 'Example/Button',
  component: Button,
  // More on argTypes: https://storybook.js.org/docs/react/api/argtypes
  argTypes: {
    backgroundColor: { control: 'color' },
  },
} as ComponentMeta<typeof Button>;

// More on component templates: https://storybook.js.org/docs/react/writing-stories/introduction#using-args
const Template: ComponentStory<typeof Button> = (args) => <Button {...args} />;

export const Primary = Template.bind({});
// More on args: https://storybook.js.org/docs/react/writing-stories/args
Primary.args = {
  primary: true,
  label: 'Button',
};

export const Secondary = Template.bind({});
Secondary.args = {
  label: 'Button',
};

export const Large = Template.bind({});
Large.args = {
  size: 'large',
  label: 'Button',
};

export const Small = Template.bind({});
Small.args = {
  size: 'small',
  label: 'Button',
};

export default 부분의 title에 Example/Button이라고 되어있는데,

/ 기호를 사용하여 뒤에 덧붙히게 되면 Storybook 화면상에서는

토글 가능한 하위 리스트 형식으로 보여지게 된다.

예를 들면, title: 'Example/Button/Hello' 라고 하게 되면

이렇게 나오게 된다.

Controls

그리고 argTypes의 경우, 위 이미지에서 controls에 어떤 컨트롤을 보여줄 지를 정할 수 있다.

Controls란, 소스코드 수정을 통해서가 아닌 사용자 입장의 브라우저 조작을 통해

컴포넌트 변화를 체감할 수 있게 해주는 일종의 플러그인이다.

Interactions

지금까지 얘기한건 UI 변화를 육안으로 확인할 수 있는 차원의 이야기였다.

프론트엔트 테스트라면 보통 Jest, Enzyme, React Testing Library를 떠올릴텐데,

감사하게도 Storybook에서는 이것들이 포함되어 있으며,

RTL로 테스트 코드를 작성하는 것과 똑같이 작성해서 테스트를 수행한다.

여기서 좋은 점이, Storybook이다 보니 컴포넌트의 변화를 시각적으로 느낄 수 있기 때문에

Storybook에서 테스트 코드를 짜면 RTL 코드 한 줄 한 줄을 애니메이션처럼 실행해서 볼 수 있다.

실용성 1도 없는 간단한 예시를 하나 보자고.

어차피 stories/Page.stories.tsx에 있다.

import { within, userEvent } from '@storybook/testing-library';
...
LoggedIn.play = async ({ canvasElement }) => {
  const canvas = within(canvasElement);
  const loginButton = await canvas.getByRole('button', { name: /Log in/i });
  await userEvent.click(loginButton);
};

play 함수를 정의하고, 구현체에 RTL에서 흔히 작성하던 코드를 거의 흡사하게 작성하였다. (사실 이것도 샘플)

그러면 Storybook 화면 하단의 Interactions 탭을 클릭하면 테스트 케이스의 PASS/FAIL 여부를 확인할 수 있다.

Conclusion

Festa라는 회사 대표가 Storybook에 대해 세미나하는 영상이 있으니 참고하자고.

link: https://youtu.be/jc9xKzdkYDg

여기서 설명 들어보면 알겠지만, Storybook을 잘 활용하기 위해서는

컴포넌트들이 '순수'해야 한다고 한다.

올바른 Storybook 사용을 위해서는 커스텀 훅이나, 어느 전역 상태를 후킹하거나 디스패치하는 행위를

컴포넌트 안에서 하는 만행을 저질러서는 안된다.
