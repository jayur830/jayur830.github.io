[Package Manager] NPM, Yarn, 그리고 Yarn v2

2023-09-02

패키지 매니저란?

  • 패키지를 다루는 작업을 편리하고 안전하게 수행하기 위해 사용되는 툴
  • 패키지: 코드의 배포를 위해서 사용되는 코드의 묶음
  • 종류: npm, yarn, pip, maven, gradle, apt, yum, homebrew 등

npm

  • Node Package Manager의 약자
  • Node.js 환경에서 만들어진 모듈을 설치하고 관리해주는 도구

장점

  • 높은 점유율: 전세계적 규모의 다운로드 수와 커뮤니티를 보유
  • CLI를 통한 손쉬운 설치
  • Node.js만 설치하면 자동으로 설치되므로 별도의 설치가 필요 없음

단점

  • 보안성: 패키지 설치 시 해당 패키지에 포함된 또 다른 패키지를 자동으로 실행하며, 또 그것을 실행할 수 있도록 허용
  • 개발환경과 운영환경의 패키지 버전 동기화가 되지 않는 버저닝 이슈 발생

yarn

  • npm을 개선하기 위해 Facebook이 만든 새로운 Package Manager

장점

  • 캐싱: 패키지 설치 시 OS 전역 어딘가에 해당 패키지를 캐싱함
    • 오프라인 모드에서도 사용 가능
    • 캐시 디렉토리 경로: ~/Library/Cache/Yarn/v6
  • yarn add react @types/react react-dom ... 와 같이 여러 패키지 동시 설치 시 각 패키지들을 병렬로 설치
    • 설치 시간 단축
  • 코드 실행 전 각 패키지별로 Checksum을 생성하여 yarn.lock 파일에 명시된 Checksum과 해시값을 비교하여 패키지들의 무결성 확인
  • yarn.lock 파일을 통한 의존성 트리 보장
    • 어떤 환경에서든 똑같은 버전의 패키지들을 설치함을 보장할 수 있음

단점

  • Node.js 5.0 이하 버전을 지원하지 않는다고 함.
  • npm과 달리 npm i yarn 또는 brew install yarn 을 통해 별도 설치 필요

npm과 yarn의 공통적인 고질적 문제

  • node_modules 디렉토리는 수 기가바이트의 디스크 용량을 잡아먹다.
  • node_modules 디렉토리는 패키지 탐색 시 트리 구조에서 탐색이 실패하면 다시 루트부터 탐색하는 비효율적인 방식으로 탐색다.
  • node_modules 디렉토리는 dependency가 중복되면 끌어올리기 기법(Hoisting)을 사용하여 중복된 dependency들을 최상위로 끌어올려 다른 패키지들과 동일선상에 놓이게 한다. 이렇게 되면 의도하지 않게 설치된 B 패키지를 조용히 사용할 수 있고, A나 C를 삭제했을 경우 B를 사용하지 못하거나 A를 사용할 수 없는 상황이 발생하는데, 이것을 **유령 의존성(Ghost Dependency)**이라고 한다.

Yarn Berry

위에서 설명한 npm과 yarn의 문제를 해결하기 위해 나온 새로운 Package Manager이다.

원래는 Yarn 2인데, 기존의 npm, yarn과는 패키지 관리 방식이 너무나도 달라 아예 새로운 Package Manager라고 취급할 필요가 있었고, 결국 Berry라는 이름을 붙이게 되었다고 한다.

Feature 1. PnP

모든 dependency를 Git, SVN과 같은 버전 관리 시스템에 포함하도록 함으로써 CI/CD 면에서 장점이 있음

  • CI의 장점: 다른 사람이 git clone 또는 svn checkout 을 하여 프로젝트를 받고 더 이상 yarn 커맨드로 패키지를 설치하는 세팅 작업을 할 필요가 없음
  • CD의 장점: 배포 환경에서도 패키지 설치를 별도로 하지 않게 되어 배포 시간 단축

단점

  • 아직 Yarn Berry의 PnP를 지원하지 않는 패키지들이 존재
  • 기존 npm, yarn에 익숙한 개발자들에게는 생소할 것이며, Yarn Berry로 마이그레이션 할 경우 트러블 슈팅 때문에 시간을 잡아먹을 수도 있음
  • webpack과 같이 이용할 경우 dev server를 킬 때 더 느릴 수 있음
    • https://github.com/yarnpkg/berry/issues/3311
  • 기존에 쓰던 라이브러리, 프레임워크와 호환성이 안맞을 경우 존재
    • 호환성 테이블: https://yarnpkg.com/features/pnp#compatibility-table
