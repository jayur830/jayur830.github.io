대부분의 기업들이 React를 쓰는 추세라 나도 이제부터 갈아타려고 공부하는데,

사실 React 자체는 예전부터 알고 있었지만 최적화라던지,

Saga나 Context API 등을 어떻게 같이 쓰는지 등등

깊게 파고들지는 않았었다.

이제부터라도 공부를 마음먹고 시작해보려 한다. 일단 React Hook에 대해서 살펴볼텐데,

React Hook 중 useCallback과 useMemo가 있다.

이 둘의 차이에 대해 끄적여볼까 한다.

일단 둘의 공통점은, 자원을 캐싱한다는거다.

미리 메모리에 저장해뒀다가 컴포넌트가 재렌더링될 때마다 꺼내쓰는거다.

마치 메모이제이션이랑 비슷하다. (찾아봤더니 진짜로 메모이제이션 패턴을 이용하여 구현되었다고 한다.)

차이라고 하면 함수를 반환하느냐, 값을 반환하느냐이다.

useCallback

먼저 useCallback에 대해 이야기해보려 한다.

코드를 보며 생각해보자.

import { useState } from "react";

const App = () => {
    const [a, setA] = useState(0);
    const [b, setB] = useState(0);
    const [c, setC] = useState(0);
    
    const add = () => setC(a + b);
    
    return (
        <>
            <input type="number" value={a} onKeyup={e => setA(e.target.value)} />
            +
            <input type="number" value={b} onKeyup={e => setB(e.target.value)} />
            <input type="button" value="=" onClick={add} />
            <span>{c}</span>
        </>
    );
};

export default App;

두 개의 피연산자를 덧셈하는 아주 심플한 계산기 앱이다.

그런데 문제가 있다.

a와 b의 값이 바뀔 때마다 App 컴포넌트는 다시 렌더링되는데,

이 때 add라는 함수 또한 다시 만들어져 메모리에 할당된다.

add라는 기능 자체는 변한게 없기 때문에

다시 만드는 건 불필요한 메모리 낭비일 뿐이다.

이걸 useCallback을 이용해 함수가 최초 한 번만 만들어지고 꺼내쓰도록 최적화해보자.

const add = useCallback(() => setC(a + b), []);

이렇게 하면 컴포넌트가 재렌더링되어도 add라는 함수는 재선언되지 않는다.

메모리에 이미 저장되어 있어 불필요한 메모리 낭비를 줄인 것이다.

근데 또 문제가 있다.

저렇게 해서 a와 b를 더한 값을 출력하려 해보면

새로 계산하지 않고 이전에 계산해둔 값이 출력된다.

함수뿐만 아니라 함수 내부에서 사용되는 값까지 모조리 캐싱이 되버린거다.

그렇다면 다음과 같이 하면 된다.

const add = useCallback(() => setC(a + b), [a, b]);

Dependency Array라고 하더라.

직역하면 의존성 배열인데,

말그대로 함수 내에서 사용되는 변수가 useCallback 두 번째 인자에 들어온 배열의 변수들의 상태에 의존한다는 거다.

즉, 함수 밖에서의 상태값이 변하면 그에 의존하여 함수 내부 상태도 같이 변하도록 되어있는 것이다.

useMemo

다음으로 useMemo에 대해서 말해보려 한다.

useCallback은 함수를 캐싱하여 반환한다.

근데 useMemo는 함수와 그 함수의 반환값 모두를 캐싱하고 함수의 반환값을 반환한다.

다음 코드를 보자.

import { useState } from "react";
import dayjs from "dayjs";

const App = () => {
    const [kstTime, setKstTime] = useState(dayjs());
    
    setInterval(() => setKstTime(dayjs()), 1000);
    
    return (
        <>
            <div>KST: {kstTime}</div>
            <div>UTC: {kstTime.subtract(-9, "hours")}</div>
        </>
    );
};

export default App;

KST는 현재시간이고 UTC는 현재로부터 9시간 전의 시간이다.

이를 실시간으로 1초마다 보여주는 앱을 만들었다.

위 코드도 문제일까? 당연히 문제가 있다.

setInterval로 1초 간격으로 콜백 함수를 계속 실행한다.

그리고 UTC 시간을 표시할 때 kstTime에서 연산하여 보여주는데,

1초마다 KST 시간의 상태가 변하여 재렌더링되고, 그에 따라 새로 연산하게 된다.

1초이기 때문에 당연히 매번 상태가 바뀔테니 연산을 새로하는게 맞지만,

만약 저 interval 값을 100 밀리세컨드로 놓고 더 빈번하게 시간체크를 하게 했다면 어떨까?

그리고 시간 포맷을 초단위나 분단위가 아닌 년/월/일/시 까지만 나타내도록 했다면 어떨까?

쌉 자원 낭비다.

그래서 나는 useMemo를 이용하여 계산된 값을 캐싱하여 사용하도록 수정하기로 한다.

import { useState, useMemo } from "react";
import dayjs from "dayjs";

const App = () => {
    const [kstTime, setKstTime] = useState(dayjs());
    const utcTime = useMemo(() => kstTime.subtract(-9. "hours"), [kstTime]);
    
    setInterval(() => setKstTime(dayjs()), 1000);
    
    return (
        <>
            <div>KST: {kstTime}</div>
            <div>UTC: {utcTime}</div>
        </>
    );
};

export default App;
